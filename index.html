⚠️ Warning: AI Voice Model is UNTRAINED

The system is currently running using randomly initialized weights because the pre-trained checkpoint file (best_model.pt) is missing.

Why is it misclassifying voices / repeating the same mistake?
Without training, the deep learning model cannot distinguish acoustic features. It outputs random/near-50% probabilities for every file. Since the suspicion threshold is set to 45%, a normal voice will consistently flag as SUSPICIOUS (~49%), while an AI voice may randomly score below 45% and flag as GENUINE.

To train the model and fix this:

# 1. Download multilingual training datasets
.venv\Scripts\python.exe datasets/download_multilingual.py --all-auto

# 2. Preprocess datasets
.venv\Scripts\python.exe datasets/preprocess_multilingual.py --all

# 3. Train the model
.venv\Scripts\python.exe backend/training/train_multilingual.py --data datasets/processed --epochs 60

Once training finishes and creates the checkpoint at backend/training/checkpoints/best_model.pt, restart the backend server.

Architecture

How VoiceClone Shield works

🧬

CNN-BiLSTM Fusion

A CNN branch reads spectral texture from mel-spectrograms; a BiLSTM branch reads prosody from MFCC sequences. Cross-attention fuses both for a final verdict.

🌐

Multilingual Training

Trained on OpenSLR, Common Voice, and KathBath for genuine Indian speech, plus MLAAD and self-generated Coqui XTTS-v2 clones for Indian-language attacks.

🔵

IBM Watsonx Layer

Granite-13B converts raw model scores into plain-English fraud explanations and GDPR/RBI-aligned compliance reports.
